# Consegna Studenti - Lezione 09

## Prima Di Compilare

Questa consegna serve a leggere una codebase reale prima di modificarla.

Non devi implementare la feature. Devi raccogliere evidenze minime:

- quali file sembrano importanti;
- cosa funziona gia';
- cosa manca;
- quale dubbio vuoi chiarire prima della UI di L10.

Errore tipico da evitare: chiedere subito all'AI di "implementare la feature" senza aver letto runtime e file.

## Input

- Questa repo starter L09.
- Demo svolta in aula.
- Template facoltativo `template/codebase-reading-notes.md`.

## Passaggi Per Risolvere L'Esercizio

1. Apri la repo.
2. Avvia l'app con `pnpm dev`.
3. Osserva il comportamento disponibile: operatore corrente, lista ticket, create incompleto.
4. Individua 3 file chiave.
5. Per ogni file, scrivi perche' ti sembra rilevante.
6. Annota 2 gap concreti.
7. Scrivi 1 domanda da portare in L10.

## Output Atteso

Puoi usare il template o una nota personale.

Formato minimo:

```txt
File chiave:
- ...
- ...
- ...

Gap:
- ...
- ...

Domanda per L10:
- ...
```

## Vincoli

- Non implementare.
- Non modificare API, Prisma o DB.
- Non chiedere all'AI una patch.
- Non aggiungere bundler, migration, auth, dashboard, allegati o notifiche.
- Non incollare dati personali, credenziali, token, API key o file `.env` in tool esterni.

## Pronto Quando

Sei pronto quando riesci a spiegare in 60 secondi:

```txt
Questi file contano perche'...
Questi gap impediscono alla feature di essere completa...
In L10 devo chiarire...
```
